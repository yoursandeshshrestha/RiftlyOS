# Agency OS — Plan 15: Messaging (DMs + Channels) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build "enough messaging" for Agency OS (PRD §5.12) — direct messages between team members, per-client/project **internal** channels, and exactly one **client↔agency** channel per client that is exposed in the portal — with real-time delivery via Supabase Realtime, unread counts, @mentions that fan out to notifications, file attachments, and message history/search. Tenant isolation is enforced by Postgres Row-Level Security driven by **conversation membership** (not org alone), proven by an RLS isolation test that shows a client can see only their own client↔agency channel and never any DM or internal channel. This is **not** a Slack rebuild: no threads, reactions, huddles, edits-history, or third-party integrations.

**Architecture:** Five new tenant-scoped tables — `conversation` (type `dm | channel`, optional `client_id`/`organization_id`, `visibility` `internal | client_shared`), `conversation_member`, `message`, `message_attachment`, `read_receipt` — every one carrying `organization_id` as the leading column of a composite index, RLS enabled, and policies built on a new SECURITY DEFINER helper `public.is_conversation_member(uuid)` that layers on top of the Plan 01 helpers `public.has_org_access(uuid)` / `public.is_agency_staff()`. Visibility is membership-driven: you only see a conversation (and its messages/attachments/receipts) if you are a member of it; clients are members **only** of their own client↔agency channel. Real-time fan-out uses Supabase Realtime `postgres_changes` on the `message` table, with RLS authorizing the subscription so a browser only receives rows it may already read (defense in depth). Unread counts are derived from `read_receipt.last_read_at` vs the latest `message.created_at` per conversation. @mentions are parsed server-side from message body, resolved to member user IDs, and delivered via the Plan 05 `emitNotification()` helper using the existing `message_received` category. Attachments live in a dedicated `message-attachments` Supabase Storage bucket; downloads use expiring signed URLs. All mutations go through Server Actions backed by the RLS-scoped Supabase server client; `service_role` is never used on the user-facing path.

**Tech Stack:** Next.js 16 (App Router, TypeScript strict) · Drizzle ORM + drizzle-kit · postgres.js · Supabase Postgres + Auth + **Realtime** + **Storage** · Tailwind + shadcn/ui · Vitest (RLS + unit tests) · Plan 05 Inngest/Resend notification pipeline (reused, not re-spec'd).

**Dependencies (assume already built; do not re-spec):**
- **Plan 01 — Foundation:** `org_type` & `app_role` enums; `organizations`, `profiles`, `memberships` tables; `public.has_org_access(uuid)` and `public.is_agency_staff()` SECURITY DEFINER functions; `custom_access_token_hook` stamping `app_role` + `org_id`; `scripts/seed.ts`; the Vitest harness `tests/helpers/db.ts` exporting `sql`, `asUser(userId, fn)`, `userIdByEmail(email)`; the role-based app shells under `src/app/(internal)/` and `src/app/(portal)/`; `src/lib/auth.ts` exporting `getSession()` and `isStaff()`; the Drizzle client `src/db/index.ts`; the browser/server Supabase clients in `src/lib/supabase/`.
- **Plan 02 — Clients & CRM:** the `client` table (`clients` in schema; `id`, `organizationId`, `name`, `accountManagerId`, …). Each client has its own `organizations` row of type `client`.
- **Plan 05 — Portal/Notifications/Audit/Skills (reused):** the `notification_category` enum (already contains `message_received`), and `emitNotification(input: EmitInput)` exported from `src/lib/notifications/emit.ts`. Messaging consumes these; it does not modify them.

```ts
// --- Plan 15: Messaging ------------------------------------------------------

// A direct message between team members, or a channel.
export const conversationType = pgEnum('conversation_type', ['dm', 'channel'])

// internal      = staff-only (DMs + per-client internal ops channels)
// client_shared = the ONE client↔agency channel exposed in the portal
export const conversationVisibility = pgEnum('conversation_visibility', [
  'internal',
  'client_shared',
])

export const conversations = pgTable(
  'conversation',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    // Tenant that owns the conversation. DMs + internal channels = the agency org;
    // the client↔agency channel = the CLIENT org (so portal RLS scopes it cleanly).
    organizationId: uuid('organization_id')
      .notNull()
      .references(() => organizations.id, { onDelete: 'cascade' }),
    type: conversationType('type').notNull(),
    visibility: conversationVisibility('visibility').notNull().default('internal'),
    // Set for per-client channels (internal client channel OR the client↔agency channel).
    // NULL for DMs and the agency-ops (non-client) channel.
    clientId: uuid('client_id').references(() => clients.id, { onDelete: 'cascade' }),
    // Display name for channels; NULL for DMs (rendered from member names).
    name: text('name'),
    createdBy: uuid('created_by').references(() => profiles.id, { onDelete: 'set null' }),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  },
  (t) => ({
    // tenant-leading composite index (PRD §9 performance rule).
    byOrgType: index('idx_conversation_org_type').on(t.organizationId, t.type),
    byClient: index('idx_conversation_client').on(t.clientId),
    // At most ONE client↔agency channel per client (enforces "exactly one" + RLS scope).
    uniqClientShared: unique('uniq_conversation_client_shared').on(t.clientId, t.visibility),
  }),
)

export const conversationMembers = pgTable(
  'conversation_member',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    organizationId: uuid('organization_id')
      .notNull()
      .references(() => organizations.id, { onDelete: 'cascade' }),
    conversationId: uuid('conversation_id')
      .notNull()
      .references(() => conversations.id, { onDelete: 'cascade' }),
    userId: uuid('user_id')
      .notNull()
      .references(() => profiles.id, { onDelete: 'cascade' }),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  },
  (t) => ({
    byOrgConv: index('idx_conv_member_org_conv').on(t.organizationId, t.conversationId),
    byUser: index('idx_conv_member_user').on(t.userId),
    uniqConvUser: unique('uniq_conv_member').on(t.conversationId, t.userId),
  }),
)

export const messages = pgTable(
  'message',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    organizationId: uuid('organization_id')
      .notNull()
      .references(() => organizations.id, { onDelete: 'cascade' }),
    conversationId: uuid('conversation_id')
      .notNull()
      .references(() => conversations.id, { onDelete: 'cascade' }),
    senderId: uuid('sender_id')
      .notNull()
      .references(() => profiles.id, { onDelete: 'cascade' }),
    body: text('body').notNull().default(''),
    // Resolved @mention recipient user IDs (string[]); drives notifications + UI.
    mentions: jsonb('mentions').$type<string[]>().notNull().default([]),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  },
  (t) => ({
    // tenant-leading; conversationId + createdAt powers the thread query + unread math.
    byOrgConvCreated: index('idx_message_org_conv_created').on(
      t.organizationId,
      t.conversationId,
      t.createdAt,
    ),
  }),
)

export const messageAttachments = pgTable(
  'message_attachment',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    organizationId: uuid('organization_id')
      .notNull()
      .references(() => organizations.id, { onDelete: 'cascade' }),
    messageId: uuid('message_id')
      .notNull()
      .references(() => messages.id, { onDelete: 'cascade' }),
    uploadedBy: uuid('uploaded_by')
      .notNull()
      .references(() => profiles.id, { onDelete: 'cascade' }),

    storagePath: text('storage_path').notNull(),
    contentType: text('content_type').notNull(),
    sizeBytes: integer('size_bytes').notNull(),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  },
  (t) => ({
    byOrgMessage: index('idx_message_attachment_org_msg').on(t.organizationId, t.messageId),
  }),
)

// Unread count = messages in the conversation with created_at > last_read_at

  },

    byOrgConv: index('idx_read_receipt_org_conv').on(t.organizationId, t.conversationId),
```

```ts
import { describe, it, expect, beforeAll, afterAll } from 'vitest'
import { sql, asUser, userIdByEmail } from '../helpers/db'

describe('tenant isolation (RLS): messaging — membership-driven', () => {
  let founder: string
  let team: string
  let clientOneUser: string
  let clientTwoUser: string

  beforeAll(async () => {
    founder = await userIdByEmail('founder@milktreeagency.com')
    team = await userIdByEmail('team@milktreeagency.com')
    clientOneUser = await userIdByEmail('user1@clientone.com')
    clientTwoUser = await userIdByEmail('user2@clienttwo.com')
  })

  afterAll(async () => {
    await sql.end()
  })

  it('a client sees ONLY the client↔agency channel they belong to', async () => {
    const rows = await asUser(clientOneUser, (tx) =>
      tx`select visibility, type from public.conversation`,
    )
    expect(rows.length).toBe(1)
    expect(rows[0]!.visibility).toBe('client_shared')
    expect(rows[0]!.type).toBe('channel')
  })

  it('a client NEVER sees DMs or internal channels', async () => {
    const rows = await asUser(clientOneUser, (tx) =>
      tx`select id from public.conversation where visibility = 'internal'`,
    )
    expect(rows.length).toBe(0)
  })

  it('a client cannot read messages from internal conversations', async () => {
    const rows = await asUser(clientOneUser, (tx) =>
      tx`select m.id from public.message m
           join public.conversation c on c.id = m.conversation_id
          where c.visibility = 'internal'`,
    )
    expect(rows.length).toBe(0)
  })

  it('a client of one tenant cannot see another client\'s shared channel', async () => {
    // client-two has no seeded conversation membership → zero rows.
    const rows = await asUser(clientTwoUser, (tx) => tx`select id from public.conversation`)
    expect(rows.length).toBe(0)
  })

  it('agency staff (founder) sees DMs, internal channels, AND shared channels', async () => {
    const rows = await asUser(founder, (tx) => tx`select visibility from public.conversation`)
    const vis = rows.map((r) => r.visibility).sort()
    // dm(internal) + internal channel + client_shared channel.
    expect(vis).toEqual(['client_shared', 'internal', 'internal'])
  })

  it('a team member sees the DM and internal channel they belong to', async () => {
    const rows = await asUser(team, (tx) =>
      tx`select type from public.conversation order by type`,
    )
    // team is a member of the DM and the internal channel (not the shared one in seed).
    expect(rows.length).toBeGreaterThanOrEqual(2)
  })

  it('a non-member staff user cannot read a conversation\'s messages it does not belong to', async () => {
    // Build a fresh internal DM that ONLY the founder belongs to, then assert the
    // team member cannot read its messages even though both are agency staff.
    const [conv] = await sql`
      insert into public.conversation (organization_id, type, visibility, created_by)
      select organization_id, 'dm', 'internal', ${founder}
        from public.organizations where type = 'agency' limit 1
      returning id`
    await sql`insert into public.conversation_member (organization_id, conversation_id, user_id)
              select organization_id, ${conv!.id}, ${founder}
                from public.organizations where type = 'agency' limit 1`
    await sql`insert into public.message (organization_id, conversation_id, sender_id, body)
              select organization_id, ${conv!.id}, ${founder}, 'private founder note'
                from public.organizations where type = 'agency' limit 1`

    const founderSees = await asUser(founder, (tx) =>
      tx`select id from public.message where conversation_id = ${conv!.id}`,
    )
    const teamSees = await asUser(team, (tx) =>
      tx`select id from public.message where conversation_id = ${conv!.id}`,
    )
    expect(founderSees.length).toBe(1)
    expect(teamSees.length).toBe(0)

    await sql`delete from public.conversation where id = ${conv!.id}`
  })
})
```

```sql
-- ── Membership helper ─────────────────────────────────────────────────────────
-- True iff the current user is a member of the target conversation. SECURITY
-- DEFINER so the lookup itself is not subject to conversation_member RLS
-- (avoids recursive policy evaluation). Mirrors Plan 01 helper conventions.
create or replace function public.is_conversation_member(target_conversation uuid)
returns boolean
language sql
security definer
stable
set search_path = public
as $$
  select exists (
    select 1
    from public.conversation_member cm
    where cm.conversation_id = target_conversation
      and cm.user_id = auth.uid()
  );
$$;

-- ── Enable RLS on all five tables ─────────────────────────────────────────────
alter table public.conversation         enable row level security;
```

```ts
// Pure @mention parsing — no DB, no server-only. Handle = local part of email,
// lowercased (e.g. founder@milktreeagency.com -> "founder").

export type MentionableMember = { userId: string; email: string }

const HANDLE_RE = /(^|[^\w@])@([a-z0-9._-]+)/gi

export function handleFor(email: string): string {
  return email.split('@')[0]!.toLowerCase()
}

export function parseMentions(body: string, members: MentionableMember[]): string[] {
  const byHandle = new Map(members.map((m) => [handleFor(m.email), m.userId]))
  const result: string[] = []
  const seen = new Set<string>()
  for (const match of body.matchAll(HANDLE_RE)) {
    const handle = match[2]!.toLowerCase()
    const userId = byHandle.get(handle)
    if (userId && !seen.has(userId)) {
      seen.add(userId)
      result.push(userId)
    }
  }
  return result
}
```

> The `(^|[^\w@])` prefix in `HANDLE_RE` ensures `founder@milktreeagency.com` (an email) does not match as a mention, because the `@` is preceded by a word character.

```ts
// Pure unread-count math — no DB. A message is unread if it is newer than the
// user's last_read watermark AND was not sent by the user.

export type UnreadMessage = { senderId: string; createdAt: Date }

export function unreadFor(
  messages: UnreadMessage[],
  lastReadAt: Date | null,
  userId: string,
): number {
  const watermark = lastReadAt ? lastReadAt.getTime() : -Infinity
  return messages.filter(
    (m) => m.senderId !== userId && m.createdAt.getTime() > watermark,
  ).length
}
```

```ts
import { createSupabaseServerClient } from '@/lib/supabase/server'
import { unreadFor, type UnreadMessage } from './unread'

export type ConversationListItem = {
  id: string
  type: 'dm' | 'channel'
  visibility: 'internal' | 'client_shared'
  name: string | null
  clientId: string | null
  unread: number
}

// List conversations the current user belongs to, each with an unread count.
// All filtering is enforced by RLS (is_conversation_member); we never pass a
// user id from app code into the WHERE clause.
export async function listConversations(currentUserId: string): Promise<ConversationListItem[]> {
  const supabase = await createSupabaseServerClient()

  const { data: convs, error: convErr } = await supabase
    .from('conversation')
    .select('id, type, visibility, name, client_id')
    .order('created_at', { ascending: false })
  if (convErr) throw convErr
  if (!convs?.length) return []

  const ids = convs.map((c) => c.id)

  const { data: msgs, error: msgErr } = await supabase
    .from('message')
    .select('conversation_id, sender_id, created_at')
    .in('conversation_id', ids)
  if (msgErr) throw msgErr

  const { data: receipts, error: rcErr } = await supabase
    .from('read_receipt')
    .select('conversation_id, last_read_at')
    .in('conversation_id', ids)
  if (rcErr) throw rcErr

  const watermark = new Map<string, Date | null>(
    (receipts ?? []).map((r) => [r.conversation_id, r.last_read_at ? new Date(r.last_read_at) : null]),
  )
  const byConv = new Map<string, UnreadMessage[]>()
  for (const m of msgs ?? []) {
    const arr = byConv.get(m.conversation_id) ?? []
    arr.push({ senderId: m.sender_id, createdAt: new Date(m.created_at) })
    byConv.set(m.conversation_id, arr)
  }

  return convs.map((c) => ({
    id: c.id,
    type: c.type,
    visibility: c.visibility,
    name: c.name,
    clientId: c.client_id,
    unread: unreadFor(byConv.get(c.id) ?? [], watermark.get(c.id) ?? null, currentUserId),
  }))
}
```

```ts
// Idempotently ensure the ONE client↔agency channel exists for a client and
// return its id. Adds the account manager + the client org's portal users as
// members. Backed by uniq_conversation_client_shared so concurrent calls are safe.
export async function ensureClientAgencyChannel(clientId: string, createdBy: string): Promise<string> {
  const [client] = await db.select().from(clients).where(eq(clients.id, clientId))
  if (!client) throw new Error('Client not found')

  const existing = await db
    .select({ id: conversations.id })
    .from(conversations)
    .where(
      and(eq(conversations.clientId, clientId), eq(conversations.visibility, 'client_shared')),
    )
  if (existing[0]) return existing[0].id

  const [conv] = await db
    .insert(conversations)
    .values({
      organizationId: client.organizationId, // CLIENT org → portal RLS scope
      type: 'channel',
      visibility: 'client_shared',
      clientId,
      name: `${client.name} ↔ Milktree`,
      createdBy,
    })
    .returning()

  const memberIds = new Set<string>([createdBy])
  if (client.accountManagerId) memberIds.add(client.accountManagerId)

  // Add the client org's portal users (role 'client') as members.
  const clientUsers = await db
    .select({ userId: memberships.userId })
    .from(memberships)
    .where(and(eq(memberships.organizationId, client.organizationId), eq(memberships.role, 'client')))
  for (const u of clientUsers) memberIds.add(u.userId)

  await db.insert(conversationMembers).values(
    [...memberIds].map((userId) => ({
      organizationId: client.organizationId,
      conversationId: conv!.id,
      userId,
    })),
  )
  return conv!.id
}
```

```ts
'use server'

import { revalidatePath } from 'next/cache'
import { randomUUID } from 'node:crypto'
import { getSession } from '@/lib/auth'
import { createSupabaseServerClient } from '@/lib/supabase/server'
import { listMembers } from '@/lib/messaging/queries'

async function requireSession() {
  const session = await getSession()
  if (!session) throw new Error('Not authenticated')
  return session
}

// Send a message. RLS (message_insert) rejects if the sender is not a member.
export async function sendMessage(conversationId: string, body: string) {
  const session = await requireSession()
  const text = body.trim()
  if (!text) throw new Error('Message is empty')

  const supabase = await createSupabaseServerClient()

  // Resolve @mentions against the conversation's members (RLS-scoped read).
  const members = await listMembers(conversationId)
  const mentioned = parseMentions(text, members)

  // Look up the conversation's org for the notification tenant tag.
  const { data: conv, error: convErr } = await supabase
    .from('conversation')
    .select('organization_id')
    .eq('id', conversationId)
    .single()
  if (convErr || !conv) throw new Error('Conversation not found or access denied')

  const { error } = await supabase.from('message').insert({
    organization_id: conv.organization_id,
    conversation_id: conversationId,
    sender_id: session.userId,
    body: text,
    mentions: mentioned,
  })
  if (error) throw error

  // Fan out @mention notifications (never notify the sender).
  for (const userId of mentioned) {
    if (userId === session.userId) continue
    await emitNotification({
      organizationId: conv.organization_id,
      userId,
      category: 'message_received',
      title: 'You were mentioned',
      body: text.slice(0, 140),
      linkPath: `/messages/${conversationId}`,
      data: { conversationId, senderId: session.userId },
    })
  }

  revalidatePath(`/messages/${conversationId}`)
  revalidatePath('/messages')
}

// Create a DM between the current staff user and one other user. Idempotent-ish:
// callers should reuse an existing DM if one exists (UI concern). RLS
// (conversation_insert) requires the creator be agency staff.
export async function createDirectMessage(otherUserId: string) {
  const session = await requireSession()
  if (session.role !== 'founder' && session.role !== 'team') {
    throw new Error('Only agency staff can start DMs')
  }
  if (!session.orgId) throw new Error('No organization on session')
  const supabase = await createSupabaseServerClient()

  const { data: conv, error } = await supabase
    .from('conversation')
    .insert({
      organization_id: session.orgId,
      type: 'dm',
      visibility: 'internal',
      created_by: session.userId,
    })
    .select('id')
    .single()
  if (error || !conv) throw error ?? new Error('Failed to create DM')

  const { error: memErr } = await supabase.from('conversation_member').insert([
    { organization_id: session.orgId, conversation_id: conv.id, user_id: session.userId },
    { organization_id: session.orgId, conversation_id: conv.id, user_id: otherUserId },
  ])
  if (memErr) throw memErr

  revalidatePath('/messages')
  return conv.id as string
}

// Create an internal channel (optionally bound to a client). Staff-only via RLS.
export async function createChannel(input: {
  name: string
  clientId?: string | null
  memberUserIds: string[]
}) {
  const session = await requireSession()
  if (session.role !== 'founder' && session.role !== 'team') {
    throw new Error('Only agency staff can create channels')
  }
  if (!session.orgId) throw new Error('No organization on session')
  const supabase = await createSupabaseServerClient()

  const { data: conv, error } = await supabase
    .from('conversation')
    .insert({
      organization_id: session.orgId,
      type: 'channel',
      visibility: 'internal',
      client_id: input.clientId ?? null,
      name: input.name,
      created_by: session.userId,
    })
    .select('id')
    .single()
  if (error || !conv) throw error ?? new Error('Failed to create channel')

  const ids = new Set<string>([session.userId, ...input.memberUserIds])
  const { error: memErr } = await supabase.from('conversation_member').insert(

      organization_id: session.orgId!,
      conversation_id: conv.id,
      user_id,
    })),
  )
  if (memErr) throw memErr

  revalidatePath('/messages')
  return conv.id as string
}

// Upsert the current user's read watermark for a conversation to "now".
export async function markRead(conversationId: string) {
  const session = await requireSession()
  const supabase = await createSupabaseServerClient()
  const { data: conv } = await supabase
    .from('conversation')
    .select('organization_id')
    .eq('id', conversationId)
    .single()
  if (!conv) throw new Error('Conversation not found or access denied')

  const { error } = await supabase
    .from('read_receipt')
    .upsert(
      {
        organization_id: conv.organization_id,
        conversation_id: conversationId,
        user_id: session.userId,
        last_read_at: new Date().toISOString(),
      },
      { onConflict: 'conversation_id,user_id' },
    )
  if (error) throw error
  revalidatePath('/messages')
}

// Issue a signed upload URL for an attachment; the object path is tenant +
// conversation prefixed so storage RLS (Task 4) enforces membership.
export async function createAttachmentUploadUrl(conversationId: string, fileName: string) {
  const session = await requireSession()
  const supabase = await createSupabaseServerClient()
  const { data: conv } = await supabase
    .from('conversation')
    .select('organization_id')
    .eq('id', conversationId)
    .single()
  if (!conv) throw new Error('Conversation not found or access denied')

  const objectPath = `${conv.organization_id}/${conversationId}/${randomUUID()}-${fileName}`
  const { data, error } = await supabase.storage
    .from('message-attachments')
    .createSignedUploadUrl(objectPath)
  if (error || !data) throw error ?? new Error('Failed to sign upload')
  return { uploadUrl: data.signedUrl, token: data.token, storagePath: objectPath }
}

// Record an attachment row after the client-side upload completes.
export async function recordAttachment(input: {
  messageId: string
  conversationId: string
  fileName: string
  storagePath: string
  contentType: string
  sizeBytes: number
}) {
  const session = await requireSession()
  const supabase = await createSupabaseServerClient()
  const { data: conv } = await supabase
    .from('conversation')
    .select('organization_id')
    .eq('id', input.conversationId)
    .single()
  if (!conv) throw new Error('Conversation not found or access denied')

  const { error } = await supabase.from('message_attachment').insert({
    organization_id: conv.organization_id,
    message_id: input.messageId,
    uploaded_by: session.userId,
    file_name: input.fileName,
    storage_path: input.storagePath,
    content_type: input.contentType,
    size_bytes: input.sizeBytes,
  })
  if (error) throw error
  revalidatePath(`/messages/${input.conversationId}`)
}

// Issue a short-lived signed download URL for an attachment.
export async function getAttachmentDownloadUrl(storagePath: string) {
  await requireSession()
  const supabase = await createSupabaseServerClient()
  const { data, error } = await supabase.storage
    .from('message-attachments')
    .createSignedUrl(storagePath, 60 * 5) // 5 minutes
  if (error || !data) throw error ?? new Error('Failed to sign download')
  return data.signedUrl
}
```

Real-time delivery uses Supabase Realtime `postgres_changes` on `public.message` (published in Task 4). The browser subscribes filtered by `conversation_id`; RLS authorizes the stream so only authorized rows arrive. The composer posts via the `sendMessage` Server Action.

- [ ] **Step 1: Write `src/lib/messaging/realtime.ts`**

```ts
'use client'
import { createSupabaseBrowserClient } from '@/lib/supabase/client'
import type { RealtimeChannel } from '@supabase/supabase-js'

export type RealtimeMessageRow = {
  id: string
  conversation_id: string
  sender_id: string
  body: string
  mentions: string[]
  created_at: string
}

// Subscribe to new messages in a conversation. RLS gates which rows the socket
// is allowed to deliver. Returns an unsubscribe function.
export function subscribeToConversation(
  conversationId: string,
  onInsert: (row: RealtimeMessageRow) => void,
): () => void {
  const supabase = createSupabaseBrowserClient()
  const channel: RealtimeChannel = supabase
    .channel(`messages:${conversationId}`)
    .on(
      'postgres_changes',
      {
        event: 'INSERT',
        schema: 'public',
        table: 'message',
        filter: `conversation_id=eq.${conversationId}`,
      },
      (payload) => onInsert(payload.new as RealtimeMessageRow),
    )
    .subscribe()

  return () => {
    void supabase.removeChannel(channel)
  }
}
```

```tsx
'use client'
import { useState, useTransition } from 'react'
import { Button } from '@/components/ui/button'
import { sendMessage } from '@/app/(internal)/messages/actions'

export function Composer({ conversationId }: { conversationId: string }) {
  const [body, setBody] = useState('')
  const [error, setError] = useState<string | null>(null)
  const [pending, startTransition] = useTransition()

  function submit(e: React.FormEvent) {
    e.preventDefault()
    const text = body.trim()
    if (!text) return
    setError(null)
    startTransition(async () => {
      try {
        await sendMessage(conversationId, text)
        setBody('')
      } catch (err) {
        setError(err instanceof Error ? err.message : 'Failed to send')
      }
    })
  }

  return (
    <form onSubmit={submit} className="mt-3 flex flex-col gap-2">
      <textarea
        className="min-h-[64px] rounded border p-2 text-sm"
        placeholder="Message…  (use @handle to mention)"
        value={body}
        onChange={(e) => setBody(e.target.value)}
        onKeyDown={(e) => {
          if (e.key === 'Enter' && !e.shiftKey) {
            e.preventDefault()
            submit(e as unknown as React.FormEvent)
          }
        }}
      />
      {error && <p className="text-sm text-red-600">{error}</p>}
      <div className="flex justify-end">
        <Button type="submit" disabled={pending}>
          {pending ? 'Sending…' : 'Send'}
        </Button>
      </div>
    </form>
  )
}
```
